# Phase 1 投资者移动端 V1 后端补齐包

状态：`active`  
适用范围：`houjinongfuai-working/backend`、`lovable-working /i`、`Phase 1`  
更新时间：`2026-04-14`

## 1. 本轮结论

投资者移动端已经在前端切换为“项目披露 + 意向登记 + 消息跟进”的 IR 形态，后端本轮需要补的是可审计、可跟进、可平滑升级的基础对象，而不是提前做成正式交易系统。

本轮冻结结论：

1. 先补 `投资者联系人`、`项目意向单`、`意向事件流`、`资料访问留痕`。
2. 所有“进入正式认购”的动作只允许表现为 `converted_offline`，不能在当前链路里伪装成真实认购单。
3. 不改变现有 `/u/scan`、`/u/session`、`/u/history`、`/ops/orders`、`/ops/sessions` 的运行语义。

## 2. 本轮纳入范围

- 投资者移动端提交基础联系资料
- 对项目或项目分区提交投资意向
- 顾问跟进状态流与原因码沉淀
- 资料室浏览/下载/分享留痕
- 面向后续接口开发的 DDL、状态机和测试骨架

## 3. 明确不在本轮

- 正式认购、支付、打款回单、结算、分红、提现
- 合格投资者认证与适当性评估
- 电子签约与协议归档
- 份额、持仓、收益曲线、税务单据
- 面向公众开放下载即可投资的募集能力

## 4. 核心对象

### 4.1 `investor_contact`

定位：投资者侧联系人主档，用于承接 `/i` 端匿名访问之后的首个可追踪身份。

关键点：

- 允许只有手机号和姓名先入库，后续再逐步补组织、职位、风险偏好。
- `source_session_key` 用于把匿名浏览期行为与后续联系人合并。
- 该对象不是正式资金账户，也不是登录态账户。

### 4.2 `investor_project_interest`

定位：投资者对项目的意向登记主单。

关键点：

- 一条意向只代表“愿意进一步沟通/了解/线下推进”。
- `intent_amount` 仅是意向金额，不得直接作为认购金额或持仓依据。
- `project_block_id` 可选，用于表达投资者是否对某个项目分区有明确偏好。

### 4.3 `investor_project_interest_event`

定位：投资意向状态变化与顾问动作的不可变事件流。

关键点：

- 必须能回放“谁在何时把线索从什么状态推进到什么状态”。
- 关闭类动作必须带 `reason_code`。
- 后续即使补 API，也要优先写事件，再更新主表快照。

### 4.4 `investor_material_access_log`

定位：资料室访问留痕。

关键点：

- 记录浏览、下载、分享等动作。
- 便于顾问判断项目热度、尽调深度和后续触达优先级。
- 只记录访问事实，不承担资料文件本体存储。

## 5. 状态口径

投资意向状态固定为：

- `submitted`
- `contacted`
- `materials_shared`
- `meeting_scheduled`
- `watchlist`
- `converted_offline`
- `closed_lost`
- `archived`

说明：

- `converted_offline` 只表示离开投资者移动端 V1，进入线下尽调、法务或合规认购流程。
- 它不表示已经成交，也不表示平台上已经形成正式订单。

## 6. 后续接口冻结草案

以下仅作为下一阶段接口冻结草案，本轮不直接修改运行中的 API：

- `POST /investor/contacts`
- `POST /investor/project-interests`
- `GET /investor/project-interests`
- `POST /investor/project-interests/:id/events`
- `POST /investor/material-access`

接口约束预留：

- 所有写入都要保留 `source_channel`。
- 所有状态推进都要同时落主表快照和事件表。
- 任何“转正式流程”都不能复用现有灌溉订单或支付订单对象。

## 7. 验收口径

满足以下条件，才算本轮后端补齐完成：

1. 一个投资者可从联系人主档追到项目意向、顾问跟进和资料访问留痕。
2. 前端 `/i` 不再依赖“假持仓、假收益”来支撑投资者转化。
3. 新对象与现有运行、订单、支付主链完全解耦，不改变原有关键路径语义。
4. 后续如果要做正式认购，只需要在 `converted_offline` 之后接新的合规模块，而不是推翻当前对象层。
