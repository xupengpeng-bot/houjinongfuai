# Phase 1 用户流优化包 v2

状态：`active`
适用范围：`houjinongfuai / lovable / Phase 1`
目标：把系统从“主链能跑”继续收成“操作顺手、理解直接、少跳页、少判断”的正式工作系统。

## 1. 本轮结论

当前已经形成 3 条主工作流：

1. 配置工作流：项目 / 区块 / 图源 / 网络模型 / 发布
2. 调度工作流：参数 / 求解预览 / 设备脚本 / ACK / 运行观测
3. 处置工作流：告警 / 工单 / 会话观测 / 审计

这轮继续确认 3 个原则：

1. 主页面必须是操作页，不是说明页。
2. 页面要以“当前对象、当前状态、当前动作”为中心，而不是堆建议、提示、引导文案。
3. 高使用频率页面要优先压平操作流，减少跳页和上下文切换。

## 6. 验收口径

当下列条件同时成立时，才算真正接近“丝滑可用”：

1. 配置人员可以沿一条主路径完成上传、分析、草稿保存、发布。
2. 调度人员可以直接看懂候选方案、风险、transport 健康和设备脚本。
3. 值守人员可以在处置台完成绝大多数告警和工单闭环。
4. 农户端用户可以顺畅完成开始、停止、查看结果。
5. 移动运维人员可以顺畅从待办进入工单并完成处理。
6. `backend build`、`backend unit test`、关键 `e2e`、`frontend build`、`frontend smoke`、`frontend lint` 全绿。

## 9. 2026-03-29 配置台与调度台顺手度收口
