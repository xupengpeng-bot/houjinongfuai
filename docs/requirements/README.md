## 使用规则

1. 业务需求文档服务于人类协作和业务确认，描述整体目标、范围、功能点、对象关系和验收口径。
2. 任何 AI 研发任务都不能替代业务需求文档本身。
3. 当业务需求未确认时，不应直接拆解执行型 AI 任务。
4. 当业务需求确认后，PM 再到开发体系目录中拆解为 typed tasks 给 AI 执行。
