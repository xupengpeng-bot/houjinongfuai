# COD-2026-03-24-005 项目物理位置行政区后端基线任务

Status: open
Audience: 软件工程师
Purpose: add a clean-slate project-side physical-location field so Asset create can default its manual physical region from the selected project without re-mixing business ownership and physical location semantics.

## Why this task exists

Current frozen model keeps two different meanings:

- `project.region_id`
  - business ownership region
  - business `region.id` UUID only
- `asset.manual_region_id`
  - manual physical-location administrative region
  - `region_reference.code` only

The new product rule is:

- Asset form must not auto-pick a physical region on its own
- when a project is selected in Asset create, the Asset physical-location administrative region should default from the project's own physical-location administrative region
- this must remain a default value only, not a forced binding

This means Project now needs its own physical-location field instead of reusing business `region_id`.
