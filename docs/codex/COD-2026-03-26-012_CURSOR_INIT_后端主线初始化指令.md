1. 任务理解
你现在进入本仓库的后端主线初始化协作阶段。
目标是在当前 Phase 1 冻结边界下，以 backend 为唯一业务真相源推进后端、迁移、测试、接口与数据范围能力。

2. 全局影响检查
你必须遵守：
- backend 是运行、计费、安全、权限、审计唯一真相
- 不把业务真相下沉到前端
- 不回到大范围方案讨论
- 不破坏当前 contract 与 Phase 1 冻结边界

3. 本次范围内
初始化阶段只做以下事情：
- 读取 docs/p1、docs/protocol、docs/uat、docs/governance 中与后端直接相关的边界
- 盘点 backend 当前模块、迁移、seed、测试、接口面
- 标出“当前主链路已完成 / 缺口 / 高风险边界”
- 后续按 Codex 派发的大任务包执行，不自行扩 scope

4. 本次范围外
不要做：
- 自行开启新产品线
- 自行扩展 Phase 2 / 3 功能
- 自行发明与主线不一致的数据模型
- 让前端承担权限、范围、计费、审计真相

5. 实施方案
默认执行规则：
- 所有交付必须带测试、迁移或可验证依据
- 先保证接口和数据真相，再谈体验
- 若需要兼容，只能做最小兼容，不改变核心规则
- 所有正式任务先留痕，再实现，再回交 Codex 复审

6. 风险与依赖
风险：
- 若后端边界漂移，前后端和 UAT 会持续返工
- 若无测试和迁移留痕，主线稳定性会快速下降
依赖：
- 后续正式任务由 Codex 派发并复审

7. 验收方式
验收标准：
- Cursor 后续按大任务包推进
- 交付具备测试或可验证依据
- 不再扩出冻结范围外逻辑

8. 需要交接给谁
初始化完成后按 Codex 后续任务包继续执行。
