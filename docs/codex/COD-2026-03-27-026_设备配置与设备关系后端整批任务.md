# COD-2026-03-27-026 设备配置与设备关系后端整批任务

Status: closed
Audience: Cursor
Purpose: deliver one larger backend baseline for the project-configuration device domain so the next frontend batch can consume one frozen contract set instead of multiple small follow-ups.

## 1. 当前阶段

- Phase 1
- project/block scope frontend closure is entering local acceptance
- next domain shifts to device configuration

## 2. 目标

In one backend batch, freeze the device-configuration contract baseline for:

1. device types
2. device ledger / device configuration
3. asset-to-device binding essentials
4. device relation / topology essentials
5. options APIs needed by frontend forms and filters

## 3. 交付范围

至少覆盖以下能力中的可交付基线：

1. `device-types`
   - list
   - detail
   - options if needed
2. `devices`
   - list
   - detail
   - create / update if current schema and safety rules permit
   - project / asset / type filter support
3. `device-relations`
   - list or graph-oriented read model
   - create / update baseline only if safe and additive
4. frontend-facing form options
   - projects
   - blocks if needed
   - assets
   - device types
   - candidate devices / relation endpoints if needed

## 4. 约束

- do not reopen architecture
- additive only; do not break current runtime / billing semantics
- device business identity remains `IMEI`
- all write paths must be auditable and conservative
- if a risky write baseline cannot be frozen safely in this batch, prefer read + options + detail first and note the pending write gap clearly

## 5. 最低完成标准

1. backend contract set is frozen enough for one frontend batch
2. routes, DTOs, and response envelopes are consistent
3. build passes
4. unit or e2e skeletons are added where practical
5. docs writeback clearly states what frontend can consume next
