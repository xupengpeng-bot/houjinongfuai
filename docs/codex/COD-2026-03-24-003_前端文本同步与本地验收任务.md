# COD-2026-03-24-003 前端 Git 拉取与本地验收任务

Status: open
Audience: 软件工程师
Purpose: define how the software engineer pulls Lovable cloud output from GitHub and completes local verification.

## Trigger condition

Run this task only after Lovable cloud changes have auto-synced to GitHub and PM has confirmed local pull verification should start.

## Input required from PM / Git

1. active frontend task id
2. frontend repo branch to pull
3. any claimed completion notes from Lovable if available

## Local software-engineer responsibilities

1. pull the latest frontend GitHub result into the local frontend repo
2. record the pulled HEAD commit SHA
3. verify the changed behavior locally
4. run local frontend build when the task requires it
5. verify the result against the frozen contract and local backend baseline
6. report whether the frontend task is locally closable

## Hard constraints

- Do not modify unrelated frontend scope during local verification.
- Do not close the frontend task until local verification passes.
- If local pulled code does not match the claimed behavior, write it down explicitly.

## Required result format

1. synced task id
2. pulled frontend commit SHA
3. changed files observed locally
4. local verification result
5. local build result
6. whether the task is closable
7. pending issue if blocked
