# COD-2026-03-26-015 区块与电力计量点后端真实契约任务

Status: closed
Audience: Cursor
Purpose: deliver the minimal real backend contract for project blocks and metering points so the new frontend shells can stop relying on mock-only placeholders.

## 1. 当前阶段

- Phase 1
- 大颗粒主线推进阶段
- 当前目标：让 `LVB-4020` 第一批前端壳体进入真实后端联调准备态

## 2. 任务目标

补齐以下最小真实后端能力：

1. `project_block`
   - list
   - detail
   - create
   - update
   - options / search
2. `metering_point`
   - list
   - detail
   - create
   - update
   - options / search

## 3. 范围

允许做：

- migration 微调
- controller / service / dto
- list / detail / create / update contract
- options / search contract
- seed 样例
- integration / e2e skeleton
- OpenAPI / Swagger 描述
- docs 和 RESULT 回写

不允许做：

- 删除现有骨架
- 重开架构
- 做复杂成本分摊算法
- 做完整 solver 算法
- 做前端业务页面

## 4. 冻结 contract 方向

### 4.1 project_block

最小读写字段：

- `id`
- `block_code`
- `project_id`
- `project_name`
- `block_name`
- `center_latitude`
- `center_longitude`
- `area_size`
- `priority`
- `status`
- `remarks`

规则：

- `block_code` 系统自动生成
- `project_id` 必填
- 列表支持按编码 / 名称 / 项目搜索

### 4.2 metering_point

最小读写字段：

- `id`
- `metering_point_code`
- `project_id`
- `project_name`
- `block_id`
- `block_name`
- `asset_id`
- `primary_meter_device_id`
- `point_name`
- `metering_type`
- `rated_capacity_kva`
- `status`
- `remarks`

规则：

- `metering_point_code` 系统自动生成
- 列表支持按编码 / 名称 / 项目搜索
- 支持项目 / 区块 / 资产 / 主表设备 options，便于前端搜索型选择器使用
