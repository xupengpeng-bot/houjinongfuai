# COD-2026-03-24-002 资产位置搜索后端基线任务

Status: open
Audience: Codex
Purpose: deliver the minimal backend baseline required before Lovable may execute `LVB-4015`.

## 1. 当前阶段

- Phase 1
- clean-slate 模型净化与收口阶段
- 当前任务只做最小搜索 contract，不扩 GIS 平台

## 2. 任务来源

- frontend task dependency: `LVB-4015-资产位置搜索与精确定位`
- PM dispatch date: `2026-03-24`

## 3. 目标

补齐 Asset manual physical location 的最小搜索 contract，使前端可以在项目业务区域约束下做位置搜索和精确定位。

## 4. 范围

允许做：

- 文档
- 最小 query contract
- 最小 REST read endpoint
- 测试骨架
- `docs/uat/lovable-codex-sync.md` 更新

不允许做：

- 全国大行政区树前端回退方案
- GIS 大屏
- 导航或轨迹系统

## 5. 冻结规则

- `project_id` 决定业务归属
- `manual_region_id` 的单一真口径是 `region_reference.code`
- 位置搜索默认受项目所属业务区域约束
- 搜索结果至少能回填：
  - `manual_region_id`
  - `manual_address_text`
- 若有坐标，则同时支持：
  - `manual_latitude`
  - `manual_longitude`

## 6. 期望交付

1. 最小搜索 endpoint 或 contract note
2. request / response 说明
3. 项目区域约束规则说明
4. 错误壳和空结果口径
5. smoke 或 integration skeleton
6. 把结果回写 `docs/uat/lovable-codex-sync.md`

## 7. 验收口径

- 前端能按项目维度发起位置搜索
- 搜索结果能稳定映射到 `region_reference.code`
- 不把业务归属和物理位置混在一个字段
