# COD-2026-03-24-004 区域基础库恢复与防回归任务

Status: open
Audience: 软件工程师
Purpose: restore the foundational nationwide administrative region library for local/shared verification environments and prevent repeated loss after reset or cleanup operations.

## Why this task exists

The project uses two different region layers:

- `region_reference`
  - foundational administrative reference library for province / city / county / town / village dropdowns
- `region`
  - business region records created and used inside the project domain

`testdata:cleanup` is allowed to delete business `region` rows, but it must not remove the foundational `region_reference` library.

Current PM verification shows the live database still contains only a sample slice in `region_reference`, not a nationwide library:

- province: 1
- city: 1
- county: 1
- town: 1
- village: 3

This means the current environment is not ready for region-library-based dropdown verification.

## Task goal

Restore the nationwide `region_reference` library in the current verification environment, prove that it survives cleanup, and write down the safe operating procedure for future reset/reseed work.

## Execute now

1. Confirm the current `region_reference` state with `npm run region-reference:verify`.
2. Restore the foundational library in the current environment.
   - preferred path: `npm run db:seed:reference`
   - if additional import invocation is required, use the existing official import path instead of hand-written SQL
3. Re-run `npm run region-reference:verify`.
4. Verify that the root-level region-library result is no longer a single-sample slice.
5. Verify that `npm run testdata:cleanup` does not remove `region_reference`.
6. Update PM-facing docs or notes if the safe reseed sequence needs to be made explicit.

## Acceptance criteria

1. `region_reference` is present after recovery.
2. Verification clearly shows the library is not limited to the current 1-province sample slice.
3. Root dropdown data for the region library is available again for frontend use.
4. `testdata:cleanup` still does not delete `region_reference`.
5. The result explicitly states the safe command sequence for shared/local verification environments.

## Hard constraints

- Do not delete or truncate `region_reference`.
- Do not treat business `region` and foundational `region_reference` as the same layer.
- Do not modify frontend business code in this task.
- Do not reopen unrelated backend feature scope.
- Prefer recovery through existing seed/import scripts over ad hoc SQL.
