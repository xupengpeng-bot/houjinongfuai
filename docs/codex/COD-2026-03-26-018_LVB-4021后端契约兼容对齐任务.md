# COD-2026-03-26-018 LVB-4021 后端契约兼容对齐任务

Status: closed
Audience: Cursor
Purpose: align backend contracts with the already-delivered `LVB-4021` frontend real-mode wiring so local verification can pass without authorizing a frontend patch wave.

## 1. 当前阶段

- Phase 1
- real 联调稳定优先
- 当前目标：优先通过后端兼容层收口 `LVB-4021` 的 real-mode 联调阻塞

## 2. 任务目标

修复 `COD-2026-03-26-017` 验收中发现的后端/前端契约不一致问题，使 `LVB-4021` 在不修改前端业务代码的前提下具备再次本地验收条件。

必须收口：

1. `project-blocks`
   - 兼容前端使用的 `area_hectare`
   - 保持后端 canonical 字段 `area_size`
   - 列表 / 详情返回壳对齐前端当前读取方式
2. `metering-points`
   - 兼容前端使用的 `point_code`
   - 兼容前端使用的 `point_type`
   - 保持后端 canonical 字段 `metering_point_code` / `metering_type`
3. `GET /metering-points/form-options`
   - 在现有 `projects / blocks / assets / devices` 基础上，补齐前端当前所需的 `point_types` / `statuses`
4. 兼容不能破坏 canonical schema，不允许把后端正式字段退化回前端临时命名

## 3. 范围

允许做：

- DTO / response mapper
- controller / service response shaping
- compatibility alias fields
- form-options 扩展
- e2e / smoke 更新
- docs / RESULT 回写

不允许做：

- 修改前端业务代码
- 推翻 `project_block` / `metering_point` canonical schema
- 重开架构讨论
- 扩 scope 到区块编辑入口、更多页面联调

## 4. 兼容收口要求

### 4.1 project-blocks

至少保证前端 real 模式下可直接消费：

- `id`
- `block_code`
- `block_name`
- `project_id`
- `project_name`
- `area_hectare`（兼容别名）
- `status`

后端内部仍以 `area_size` 为 canonical 字段。

### 4.2 metering-points

至少保证前端 real 模式下可直接消费：

- `id`
- `point_code`（兼容别名）
- `point_name`
- `project_id`
- `project_name`
- `block_id`
- `block_name`
- `point_type`（兼容别名）
- `rated_capacity_kva`
- `status`

后端内部仍以 `metering_point_code` / `metering_type` 为 canonical 字段。

### 4.3 form-options

`GET /metering-points/form-options` 需至少返回：

- `projects`
- `blocks`
- `assets`
- `devices`
- `point_types`
- `statuses`

其中 `point_types` / `statuses` 需直接可供前端下拉使用，用户侧文案为中文。

## 5. 验收口径

1. 前端 `LVB-4021` 无需额外改业务代码即可再次进入 real-mode 验收
2. backend build 通过
3. 兼容层不破坏 canonical schema
4. e2e / smoke 至少覆盖一条 blocks 和一条 metering-points 的兼容字段断言
5. 明确说明是否已 ready for `COD-2026-03-26-017` re-verify
