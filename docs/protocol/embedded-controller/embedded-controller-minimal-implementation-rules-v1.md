# 嵌入式控制器最小实现规则 v1

## 1. 目标

这份文档用于约束“根据规范生成的程序必须尽可能精简、稳定”。

## 2. 精简规则

### 2.1 未启用模块不编译业务逻辑

- 未在任务书中启用的模块，不生成业务代码
- 未在配置中启用的模块，不进入运行时流程

### 2.2 未使用字段不出现在报文中

- 未参与当前报文语义的字段不组包
- 未参与当前产品能力集的字段不保留

### 2.3 未定义管脚不生成控制逻辑

- 任务书和管脚映射未定义的接口，不允许 AI 自动猜测实现

### 2.4 不做无用抽象

- 小型板卡禁止为一两个动作引入过重抽象层
- 只保留能复用且稳定的抽象

## 3. 稳定规则

### 3.1 固定缓冲区优先

- 协议组包优先用固定 buffer
- 避免大量动态内存

### 3.2 幂等优先

- 所有动作执行都要能按 `correlation_id` 去重

### 3.3 关键累计量优先持久化

必须优先保证：

- `cumulative_runtime_sec`
- `cumulative_energy_wh / cumulative_energy_kwh`
- `cumulative_flow`

### 3.4 调试日志与主流程解耦

- 串口日志不能阻塞发包主流程

## 4. AI 生成代码前的检查清单

- 是否已有产品模板选型
- 是否有硬件能力定义
- 是否有管脚功能映射
- 是否有轻量协议规则
- 是否有 4G 发包规则
- 是否有能力集定义

缺任何一个，都不建议直接让 AI 写完整固件。
