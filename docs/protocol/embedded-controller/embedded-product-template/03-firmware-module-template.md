# 新产品固件模块模板

## 1. 文档目的

用于把产品能力拆成可开发、可测试、可复用的固件模块。

不要按“页面”“报文”“文件夹习惯”拆模块，要按职责拆。

## 2. 固件总架构

建议至少按下列层次组织：

- `bsp`
- `hal`
- `runtime`
- `connectivity`
- `protocol`
- `config`
- `workflow`
- `module-*`
- `diag`

## 5. 模块边界规则

- 一个模块只对自己的职责负责，不越层直接操作别的模块内部状态。
- `workflow` 不直接写硬件驱动，只调模块接口。
- `protocol` 不做业务决策，只做消息理解与转发。
- `config` 不直接承担业务流程，只负责配置合法性和应用。
- 模块对外暴露的状态字段必须稳定，不能临时改名。

## 6. 复用判断

每个模块都要写清楚：

- 可完全复用
- 可部分复用
- 需要重写

对应原因也要写清楚：

- 硬件差异
- 协议差异
- 能力差异
- 性能差异
