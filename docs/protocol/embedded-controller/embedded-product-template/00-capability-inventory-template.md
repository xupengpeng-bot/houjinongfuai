# 新产品能力清单模板

## 1. 文档目的

这份文档是新产品开发前最先要冻结的文档。

它解决三个问题：

- 新产品到底需要哪些能力
- 哪些能力属于 MVP，哪些属于后续版本
- 每项能力分别依赖哪些硬件、固件、平台和协议约束

## 3. 能力冻结原则

- 先按能力模块冻结，不按“页面功能”冻结。
- 一项能力必须同时说清楚控制、采集、状态、配置、告警五个面。
- 没有资源依托的能力，不允许进入 MVP。
- 平台无法承接配置和状态的能力，不允许进入 MVP。
- 无法定义验收口径的能力，不允许进入 MVP。

## 4. 能力分类建议

建议至少按以下分类盘点：

- 通用系统能力
- 控制能力
- 采集能力
- 业务流程能力
- 运维与调试能力
- 平台对齐能力

这份文档满足以下条件，才能进入下一阶段：

- 所有 MVP 能力都完成编码和归属分类
- 每项 MVP 能力都能映射到硬件资源
- 每项 MVP 能力都能映射到固件模块
- 每项 MVP 能力都能写出至少一个验收口径
- 本期不做项已经明确写出，避免后续反复插入
