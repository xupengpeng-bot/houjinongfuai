# 新产品状态机模板

## 1. 文档目的

用于定义：

- 设备生命周期状态
- 联网状态
- 业务工作流状态
- 模块状态

重点不是画图，而是冻结：

- 允许哪些状态
- 允许哪些跳转
- 哪些条件下拒绝动作

## 2. 状态机总览

建议至少拆成四类状态机：

- 板级生命周期状态机
- 联网状态机
- 业务工作流状态机
- 关键模块状态机

## 3. 板级生命周期状态机

| 状态 | 含义 | 进入条件 | 退出条件 |
| --- | --- | --- | --- |
| `booting` | 上电启动中 | 上电/复位 | 初始化完成 |
| `self_check_failed` | 自检失败 | 核心自检失败 | 故障解除/重启 |
| `idle` | 空闲待命 | 初始化完成且未运行 | 进入业务会话 |
| `running` | 运行中 | 启动成功 | 停止/异常 |
| `fault` | 故障态 | 关键故障触发 | 人工恢复/自动恢复 |

## 4. 联网状态机

| 状态 | 含义 | 进入条件 | 退出条件 |
| --- | --- | --- | --- |
| `network_init` | 网络初始化 | 开机后 | 模组准备完成 |
| `connecting` | 连接中 | 发起连接 | 建连成功/失败 |
| `online` | 在线 | 心跳正常 | 掉线/超时 |
| `offline` | 离线 | 连接中断 | 重连成功 |
| `degraded` | 降级运行 | 网络异常但本地可运行 | 网络恢复/停止 |

## 5. 工作流状态机

### 5.1 状态表

| 状态 | 含义 | 是否允许启动 | 是否允许停止 | 是否允许查询 |
| --- | --- | --- | --- | --- |
| `not_ready` | 未就绪 | `N` | `N` | `Y` |
| `ready_idle` | 就绪空闲 | `Y` | `N` | `Y` |
| `starting` | 启动中 | `N` | `Y` | `Y` |
| `running` | 运行中 | `N` | `Y` | `Y` |
| `pausing` | 暂停中 | `N` | `Y` | `Y` |
| `paused` | 已暂停 | `Y` | `Y` | `Y` |
| `stopping` | 停止中 | `N` | `N` | `Y` |
| `fault_stop` | 故障停机 | `N` | `N` | `Y` |

### 5.2 ready 判定

ready 至少要覆盖：

- 主身份可用
- 核心模块可用
- 必要配置已生效
- 供电状态满足要求
- 关键外设无致命故障

## 9. 验收关注点

- 状态名是否和平台展示一致
- 跳转条件是否可被代码直接实现
