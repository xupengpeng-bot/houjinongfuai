# 管网工程图源制图与导入规范（v1）

本文档约定 **CAD / GeoJSON 工程图** 的图层、几何与属性规则，使导出或转换后的数据能被后端 **管网工作台（network-workbench）** 稳定识别。实现逻辑以 `backend/src/modules/network-workbench/network-workbench.service.ts` 中 `buildGraphDraftFromFeatureCollection` 为准；规范修订时请同步对照该处代码。

---

## 1. 业务前提：一文件一地块（单管网区域）

- **一份图源文件（DWG / GeoJSON / 侧车 JSON）只表示一块用地内的管网**，不在同一文件内混画多块互不连通的「地块」管网。
- 若历史上使用「地块 / block」做数据隔离，产品上可收敛为：**一个项目下一套默认地块对应一套图源**；制图侧只需保证 **文件语义单一**，无需在图内再划分地块图层。
- **禁止**在同一张管网平面图里用远隔且无拓扑关系的几何体硬塞进同一导入（会导致管段无法匹配端点、预览失败）。

---

## 2. 坐标系与单位

| 项 | 要求 |
|----|------|
| GeoJSON | 遵循 RFC 7946：**坐标顺序为 `[经度, 纬度, 可选高程]`**（**longitude, latitude**）。 |
| CAD 导出 | 使用 **WGS84 经纬度** 导出为 GeoJSON 时，与上条一致。若 CAD 为平面坐标，需在导出前转换到 WGS84，否则导入后地图无法对齐。 |
| 高程 | 可选；属性中可使用 `altitude` / `elevation`，或 Point 第三维坐标。 |

---

## 3. 图层命名规范（强烈建议固定）

后端通过要素属性中的 **图层名**（见第 5 节）与 `layer_mapping` 六大角色对齐；**推荐**在 CAD 中直接使用下表 **推荐图层名**，导入时可减少手工映射。

| 角色 | `layer_mapping` 字段 | 推荐图层名（示例） | 自动识别关键词（图层名子串，不区分大小写） |
|------|----------------------|-------------------|---------------------------------------------|
| 井 | `well_layer` | `NW-WELL` / `管网-井` | `well`、`井` |
| 泵站 / 泵 | `pump_layer` | `NW-PUMP` / `管网-泵` | `pump`、`泵` |
| 阀 | `valve_layer` | `NW-VALVE` / `管网-阀` | `valve`、`阀` |
| 管段 | `pipe_layer` | `NW-PIPE` / `管网-管线` | `pipe`、`line`、`管` |
| 出水口 / 计量等 | `outlet_layer` | `NW-OUTLET` / `管网-出水` | `outlet`、`meter`、`出水` |
| 传感器 | `sensor_layer` | `NW-SENSOR` / `管网-传感` | `sensor`、`probe`、`传感` |

**制图约定：**

- 图层名使用 **UTF-8** 保存；避免同一语义建多个别名图层（如「管线」「管道」「PIPE」混用），建议统一为一套命名表。
- **节点类实体**（井、泵、阀等）与 **管段类实体** 必须分图层，便于自动推断类型。
- 装饰、图框、标题栏、非管网矢量请放在 **独立图层**（名称避免包含上表关键词），以免被误判为管段或节点。

---

## 4. 几何类型（必须满足）

### 4.1 节点（井、泵、阀、出水、传感器等）

| 几何类型 | 说明 |
|----------|------|
| **Point** | 优先；点位即设备位置。 |
| **Polygon** | 允许；后端取多边形环上各顶点 **经纬度平均值** 作为代表点。 |

以下类型 **不会** 作为节点入库：`LineString`、`MultiLineString`（会走管段逻辑）、其他类型。

### 4.2 管段

| 几何类型 | 说明 |
|----------|------|
| **LineString** | 优先；取 **第一个与最后一个折点** 作为起讫，与节点匹配。 |
| **MultiLineString** | 允许；当前实现仅使用 **第一条线串** 的起讫点（请避免多条分离管线画在同一 MultiLineString 内）。 |

**重要：** 管段两端需在 **距离上可匹配** 到已识别节点，否则该管段会被 **丢弃**（无 `from_node_code` / `to_node_code`）。建议：

- 管线端点与井/阀等 **Point 重合或足够近**；或  
- 在属性中显式填写 `from_node_code`、`to_node_code`（见第 5 节）。

### 4.3 导入成功的最低图元条件

当前后端要求：解析后 **同时** 存在 **至少 1 个节点** 与 **至少 1 条管段**，才会生成可保存的 `graph_draft`。仅有节点或仅有管线都会导致预检失败。

## 5. 要素属性（properties）约定

导出为 GeoJSON 时，每个 `Feature` 的 `properties` 建议包含图层信息；后端读取图层名的字段（**任一命中即可**）：

`layer`、`Layer`、`layer_name`、`LayerName`、`cad_layer`、`group`、`category`

### 5.1 节点编码与类型（可选但推荐）

| 含义 | 建议属性键（按优先级尝试） |
|------|---------------------------|
| 节点唯一编码 | `node_code`、`device_code`、`asset_code`、`code`、`name`、`id`、`EntityHandle` |
| 节点类型 / 角色 | `node_type`、`role`、`device_role`、`asset_type`、`object_type`、`kind`、`SubClasses` |

说明：

- 若未填编码，后端会用 `sanitize` 规则生成占位编码（如 `well_1`），不利于与现场对号。
- `SubClasses` 等为 CAD 导出常见字段，可保留。
- 若属性里的类型文本包含 `circle`、`point`、`insert` 等 CAD 类名，后端会 **更多依赖图层名** 推断角色。

### 5.2 管段编码与端点（强烈推荐）

| 含义 | 建议属性键 |
|------|------------|
| 管段编码 | `pipe_code`、`code`、`name`、`id`、`EntityHandle` |
| 起点节点编码 | `from_node_code`、`from_code`、`start_node_code`、`start_code` |
| 终点节点编码 | `to_node_code`、`to_code`、`end_node_code`、`end_code` |
| 管型 / 角色 | `pipe_type`、`role`、`kind` |
| 长度、管径 | `length_m` / `length`，`diameter_mm` / `diameter` |

未填端点编码时，后端用 **几何最近邻** 在阈值内吸附节点；图面稍有偏差可能导致管段被丢弃，**优先用属性绑定端点**。

### 5.3 其他

- `asset_id`：可选，写入节点业务资产 ID。
- `altitude` / `elevation`：可选高程。

## 6. DWG 与侧车文件

| 方式 | 说明 |
|------|------|
| **原生 DWG** | 依赖主机安装 `dwgread`（LibreDWG）转 GeoJSON；版本过新或图元复杂时可能失败。 |
| **侧车 GeoJSON / JSON** | 与 DWG **同名**、同目录：`xxx.geojson`、`xxx.import.json` 等（以后端 `resolveSourceFile` 解析规则为准）。 |
| **纯 GeoJSON 图源** | 可直接以 `.geojson` 作为图源上传，无需 DWG。 |

侧车 JSON 可包含 `FeatureCollection`、`feature_collection`、`geojson` 或 `layers` 数组等结构，以实际后端解析为准；**制图规范仍以 FeatureCollection + 本节几何与属性为准**。

---

## 7. 制图自检清单（导出 / 提交前）

- [ ] 全图仅表达 **一块区域** 的管网，无多地块混用。
- [ ] 坐标为 **WGS84 经纬度**，GeoJSON 坐标序为 **`[lon, lat]`**。
- [ ] 井/泵/阀/出水/传感器为 **Point**（或 Polygon），管段为 **LineString**（或 MultiLineString 单线串语义）。
- [ ] 管段端点与节点 **重合或足够近**，或已填 **from_/to_node_code**。
- [ ] 图层名符合 **第 3 节** 或与关键词可对应，且装饰图层不含误触发关键词。
- [ ] 图中 **至少一条管段、至少一个节点**。
- [ ] 图层名、属性中文 **UTF-8**；避免非法控制字符。

## 9. 附录：推荐图层列表示例（可直接发给制图方）

```
NW-DIM        尺寸标注（无管网关键词）
NW-FRAME      图框图签
NW-WELL       井
NW-PUMP       泵
NW-VALVE      阀
NW-PIPE       管段
NW-OUTLET     出水 / 计量点
NW-SENSOR     传感器
```

将 `NW-` 前缀换为贵司图例代号即可，但需保持 **与上表角色一一对应** 且 **图层名可被导出到 GeoJSON 的 properties**（取决于 CAD 导出插件对图层的写字段）。
