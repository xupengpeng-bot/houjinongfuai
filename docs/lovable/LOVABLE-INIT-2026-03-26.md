1. 任务理解
你现在进入本仓库的初始化协作阶段。
目标不是立刻扩需求，而是先在当前 Phase 1 冻结边界下，把前端职责、交付方式、与 Codex / Cursor 的协作边界统一下来。

2. 全局影响检查
你必须遵守：
- 前端只消费后端 API，不直接读业务表
- 不自行发明接口、字段、状态、权限规则
- 不把计费、运行、安全、权限真相下沉到前端
- 所有页面、文案、交互必须服从当前 Phase 1 冻结范围

3. 本次范围内
初始化阶段只做以下事情：
- 读取当前 docs/governance、docs/p1、docs/uat 中与前端直接相关的冻结边界
- 盘点当前前端已有页面、缺口、与后端接口映射状态
- 形成“可直接接手的前端任务清单”
- 后续按 Codex 派发的大任务包执行，不自行扩 scope

4. 本次范围外
不要做：
- 擅自新增页面范围
- 擅自扩展业务模块
- 擅自重做设计系统
- 擅自引入前端本地真相层

5. 实施方案
默认执行规则：
- 所有交付先给“改了什么 / 没改什么 / 影响什么 / 如何验证”
- 若任务涉及接口，必须明确真实请求路径、参数、返回口径
- 若发现后端缺口，只记录并回交，不自行脑补
- 所有提交必须优先可联调、可构建、可复审

6. 风险与依赖
风险：
- 如果继续让前端承担业务真相，会持续制造联调错位
- 如果只做页面壳不对齐接口，会放大返工
依赖：
- 以后所有正式任务由 Codex 派发并复审

7. 验收方式
验收标准：
- Lovable 后续按大任务包执行
- 所有交付可被 Codex 直接复审
- 不再出现“只报摘要、不落代码”的情况

8. 需要交接给谁
初始化完成后按 Codex 后续任务包继续执行。
