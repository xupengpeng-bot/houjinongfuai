## 一、执行摘要

杭州东骏科技已形成 **“智能硬件终端 + 物联网与数据中台 + 专业化运维 + 县域复制运营”** 的综合能力。在 **国家高标准农田建后管护** 与 **农业节水灌溉** 政策叠加背景下，公司拟以 **AI 与数据智能** 为新一代核心能力，打造 **面向全国、可规模复制** 的 **智慧农业灌溉与农井运营体系**，目标是在 **可靠性、可计量、可考核、可服务** 四个维度上达到国内领先水平。

**与单纯硬件销售或单一 SaaS 的差异**：项目以 **长期运营权 + 持续服务 + 数据与 AI 闭环** 为商业模式，通过 **智能终端改造、远程监管、AI 辅助运维、预测性维护、标准化巡检与区域调度**，提升机井与泵站系统完好率、缩短故障响应时间、降低农户综合灌溉成本，并为政府提供 **可量化、可追踪、可考核** 的管护与运维工具。

**区域与产业策略**：灌溉系统高度依赖 **水泵、阀门、电气控制与管材** 等部件质量与供应链响应。公司将 **自主研发与标准制定** 相结合，重点与 **泵阀与流体机械产业带、智慧水利装备集群** 建立 **联合选型、联合检测、备件联储与质量追溯** 合作，缩短交付周期、降低全寿命周期成本（详见第四章）。

**落地路径建议**：**总部迁移 + 样板乡镇试点 + 分级改造 + 逐季复制**；同步建设 **管护制度嵌入、运维 SLA、智慧农服产品** 三位一体的县域样板，再向多县扩展。

## 二、战略愿景：AI 驱动的全国先进智慧农业灌溉

### 2.1 愿景表述

构建 **“感知—认知—决策—执行—评估”** 闭环的灌溉基础设施运营体系：以机井与小型泵站为切入口，向上延伸至 **管网调度、墒情与水权约束下的灌溉策略**，向下贯通 **农户侧扫码用水、远程启停与计量结算**，以 **AI 与大模型** 提升 **故障诊断、调度优化、客服与农艺建议** 的效率与一致性，形成 **全国领先的智慧农业灌溉运营平台** 能力基座。

### 6.3 管网调度与先进算法

#### 6.3.1 「管网调度」在业务中的分层

| 层次 | 内容 |
|------|------|
| **资产与拓扑** | 管网 **图模型**（井、泵、阀、管段及拓扑关系）用于态势展示、影响分析与联控范围界定。 |
| **运行调度（泵阀联控）** | 在单套或多套 **井—泵—阀** 关系上，生成 **启停顺序、延时与运行时长** 的可执行步骤，并叠加 **设备在线、指令队列与互锁** 等约束。 |
| **联合调度（规划能力）** | 多泵站、多支管、分时段负荷与电价 / 配额下的 **全局或分区** 策略；需与水力模型或实测标定耦合，属 **分期建设** 能力。 |
| **外勤调度** | 抢修 / 巡检车辆与人员的 **路径与时序**（与上表「调度与路径」一致）。 |

#### 6.3.2 当前平台已落地的算法范式（可对外如实表述）

与自研 **调度求解（Solver）** 一致的做法是：

1. **有限候选枚举**  
   对泵阀启动类场景，在离散策略集合上展开候选，例如：**先阀后泵（valve_first）**、**先泵后阀（pump_first）**、**泵阀并行启动（simultaneous）** 等（具体以配置与关系类型为准）。

2. **多准则决策（MCDM）+ 加权线性评分**  
   对每个候选方案在 **安全、可靠性、吞吐能力、操作简易性** 等维度给出分项得分，再按运营目标选择 **权重向量**，作 **加权求和** 得到总分并排序选优。支持的目标偏好包括：**均衡（balanced）**、**稳定性优先（stability_first）**、**吞吐优先（throughput_first）**、**设备安全优先（equipment_safety）** 等。  
   该方法在学术与工程上对应 **多属性效用模型 / 加权评分决策**，特点是 **可解释、易审计**，适合灌溉现场对 **安全规程** 要求高的场景。

3. **规则化修正与运行时门禁**  
   结合 **拓扑关系类型**（如带延时的顺序互锁）、**计划运行时长** 等对分项做启发式修正；并与 **设备在线状态、活跃会话、指令队列压力** 等运行时信号结合，对 **不可派工** 情形输出 **阻断原因（blockers）** 与告警，避免「有方案但现场不能执行」。

4. **与管网图的前置条件**  
   生成上述计划前，依赖已 **发布的管网图版本**（节点、管段规模满足契约）及有效的 **泵—阀—井关联**；否则求解器返回明确说明，引导先完成 **拓扑建模与发布**。

> **说明**：上述实现是 **离散组合 + 多目标加权** 的 **运行规程优化**，不是用连续变量做 **全管网流体 CFD**；对外可称为 **「基于拓扑与运行约束的泵阀联控多目标优化决策」**。

#### 6.3.3 面向大规模管网联合调度的算法路线图（分期引入）

在具备 **水力参数标定、关键断面传感、仿真与实测闭环** 后，可逐步叠加下列 **行业成熟算法**，支撑 **全国先进** 定位下的 **联合调度与水锤防护**：

| 方向 | 典型算法或方法 | 典型用途 |
|------|----------------|----------|
| **稳态水力校核** | Hardy–Cross、Newton–Raphson（节点法 / 环方程） | 已知管径、阀门开度下的 **压力与流量分布** |
| **瞬态与水锤** | 特征线法（MOC）、简化 Joukowsky 关系 | **启停冲击**、关阀水锤评估，反哺启停规程 |
| **组合离散优化** | 混合整数线性 / 二次规划（MILP / MIQP）、分支定界 | 多泵多阀 **启停状态**、分时段 **能耗或成本** 优化 |
| **滚动调控** | 模型预测控制（MPC） | 在约束下滚动优化未来若干步的 **设定值或阀门开度** |
| **不确定性** | 随机规划、鲁棒优化、场景树 | 来水、负荷、电价不确定下的 **稳妥运行带** |
| **图上的路径与隔离** | Dijkstra / A*、最小费用流、最大流 | **冲洗路径、应急关阀、阀组隔离方案** 搜索 |
| **学习型策略（谨慎）** | 高保真仿真器 + 强化学习 / 模仿学习 | 在 **硬安全护栏** 内学习分时策略，需充分验证 |
| **外勤与资源调度** | Clarke–Wright 节约算法、车辆路径问题（VRP）启发式、列生成 | **抢修 / 巡检** 班组与车辆路径优化 |

#### 6.3.4 对外沟通建议（诚信边界）

- **现阶段**：突出 **多目标可解释决策**、**与物联网运行时门禁一体化**、**与发布管网拓扑一致**。  
- **规划阶段**：用 **「水力仿真 / 优化 / 控制」分层路线图** 表述，明确 **试点标定 → 仿真校核 → 小范围闭环 → 区域推广**，避免将 **路线图能力** 与 **已上线能力** 混为一谈。

**原则**：AI 与优化算法 **不替代** 安全联锁与法定计量检定流程；关键控制与计费逻辑 **可审计、可回滚**。

## 七、管护体系（完整资料）

**定义**：**管护** 指建后资产 **确权清晰、责任到人、制度可执行、过程可追溯、结果可考核** 的全链条管理，涵盖政府监管、村集体协调、农户参与与运营方履约。

### 7.1 管护对象与边界

- **对象**：机井、泵站、出水口、管网附属设施、计量与电控设备、通信与供电线路相关段落。  
- **边界**：与 **高标准农田项目区、灌区、行政村** 管理边界对齐；明确 **政府监管责任、村集体协管责任、运营方服务合同责任、农户合理使用义务**。

### 7.2 制度与流程（建议县域样板标配）

1. **资产台账数字化**：一井一档（位置、建设年代、泵型、管径、权属、改造记录、合同与收费标准）。  
2. **分级管护标准**：**日常巡查、定期保养、故障抢修、季节性巡检（春耕 / 夏灌前后）** 四表合一。  
3. **问题闭环**：报修入口（电话 / 小程序 / 村级联络员）→ 工单 → 到场时效 → 维修记录 → 农户确认 → 政府侧可查。  
4. **考核 KPI（示例）**：设备完好率、故障响应时间、一次修复率、投诉率、收费纠纷率、数据完整率。  
5. **政府端产品**：大屏与报表 **按乡镇 / 村 / 单井** 穿透；支持 **督导清单导出** 与 **整改跟踪**。

### 7.3 与 AI 的结合

- 用 **异常趋势** 支撑 **“预防性管护”**，将部分 **事后抢修** 转为 **计划性维护**。  
- 用 **NLP 月报** 降低基层填报负担，保证 **上级督查材料** 及时、一致。

## 八、运维体系（完整资料）

**定义**：**运维** 指保障系统 **7×24 或灌溉季高强度可用** 的工程化能力，包括 **监控、告警、工单、备件、人员、质量与安全管理**。

### 8.1 运维架构

- **一级（远程运维中心）**：全网监控、告警研判、工单派发、知识库、数据运营。  
- **二级（县域 / 区域服务中心）**：备件仓、质检与再制造工位、培训、重大故障现场指挥。  
- **三级（乡镇驻点 / 合作网点）**：巡检、抢修、村级联络，按 SLA 覆盖半径布局。

### 8.2 核心能力模块

| 模块 | 内容 |
|------|------|
| **监控与告警** | 在线率、电压电流、启停记录、通信状态；告警合并与抑制策略。 |
| **工单与 SLA** | 分级响应时限；超时升级；与客户满意度回访联动。 |
| **备件与供应链** | 与第三章产业带协同；安全库存模型；逆向物流（旧泵检测）。 |
| **质量与安全** | 带电作业规范、有限空间作业（如有）、灌溉季应急演练。 |
| **版本与 OTA** | 控制器固件灰度升级；回滚策略；变更审计。 |

### 8.3 运维数据与 AI

- 沉淀 **工单 + 设备日志 + 部件更换记录** 作为 **监督学习与规则优化** 燃料。  
- **根因分析辅助**：同类故障聚类，推动 **设计或选型迭代**。

## 九、智慧农服体系（完整资料）

**定义**：**智慧农服** 指在 **灌溉这一强刚需场景** 上，向农户与合作社提供 **好用、公平、透明、增值** 的数字化服务，并延伸至 **农艺、农资、金融（合规前提下）** 等生态合作。

### 9.1 农户侧产品（示例）

- **用水服务**：扫码 / 卡控启停、用水量与费用透明查询、电子凭证。  
- **报修与咨询**：一键报修、进度查询；常见问题智能问答（大模型 + 灌溉知识库）。  
- **灌溉提醒**：结合气象与农时，推送 **建议灌溉窗口**（非强制，遵守当地水资源管理）。  
- **合作社 / 种植主体版**：多地块、多账户、分摊结算、用水报表导出。

### 9.2 基层服务组织赋能

- **村级协管员 / 管护员 App**：巡检任务、拍照上传、离线缓存、与工单打通。  
- **农服专员工作台**：客户画像、设备档案、服务记录，支撑 **精准回访与增值销售**（如节能改造套餐）。

### 9.3 与管护、运维的闭环

- 农户侧 **评价与投诉** 回流运维 KPI；  
- 运维 **更换部件与能效改善** 反哺农服 **“省心灌溉”** 品牌叙事；  
- 政府 **考核数据** 与农服 **透明度** 共同降低 **基层协调成本**。

## 十、商业模式与实施路径（与原稿对齐）

- **核心**：持续运营权 + 服务收费（含计量、包年、分级改造摊销等，以实际合同为准）+ 数据增值服务（在政府授权范围内）。  
- **路径**：试点先行 → 标准固化 → 区域扩面 → 全面复制；首期完成 **资产摸底、逐井评估、轻重改造分类、收费与村级协调、平台接入**。  
- **融资**：企业自投 + 政策协同 + 金融配套；中后期平台公司、产业基金、政策性工具等组合。（同原稿。）

- **经济效益**：稳定需求 + 可量化收费 + 可持续服务 + 可复制扩张；沉淀税收、研发、固投与就业，带动 **安装施工、设备维护、仓储备件、智慧农服** 等周边业态。

## 十四、结论

杭州东骏科技具备 **项目经验、技术路线与市场化经营基础**；在 **AI 与数据智能** 加持下，项目可从 **“机井物联网”** 升级为 **“全国先进的智慧农业灌溉运营基础设施”**，以 **管护制度化、运维工程化、农服产品化** 形成差异化壁垒。总部迁入目标县域后，有望形成 **“企业总部 + 项目平台 + 县域试点 + 复制推广”** 格局，兼顾 **招商价值** 与 **高标准农田建后管护现实解题**，建议作为 **重点洽谈与推进项目** 开展对接。
